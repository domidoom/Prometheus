      How to use Warden
      
    

    


      

        

          
        

        

          
This is not a chatbot.

          
Warden is an autonomous agent system. It responds best to actionable requests — things you want done, not things you want to talk about.

        

        Read the full guide →
      


      
01 The agents

      

        
          

            
          

          

            
Atlas browser + research

            
Opens URLs, takes screenshots, fetches HTML, clicks buttons, reads pages.

          

        
        
          

            
          

          

            
Byte compute

            
Runs python3 / bash / node one-liners and hands back stdout.

          

        
        
          

            
          

          

            
Dexter time + scheduling

            
Date/time questions and recurring or one-shot scheduled tasks.

          

        
        
          

            
          

          

            
Iris email

            
Inbox, unread counts, search, draft, and send (if enabled).

          

        
        
          

            
          

          

            
Artemis read-only auditor

            
Reads files, greps, reviews code or conversation. Also seats The Council.

02 How to call The Council

      

        
⌘

        

          
Use the words Convene The Council on this: followed by a self-contained question. The seats can't see chat history, so include everything they need.

          
Convene The Council on this: should I learn Rust or Go first given I already know Python and want to build backend services?

          
Takes 1–3 min, 3–10× tokens. Runs in the background — Warden replies with an interim note, then pushes the verdict as a follow-up.

        

      


      
03 Asking Atlas

      

        
@

        

          
Say Delegate to Atlas: and describe the task. Multiple Atlas calls in the same turn run in parallel.

          
Delegate to Atlas: open example.com and take a screenshot.
In parallel, delegate to Atlas: (1) screenshot example.com, AND
(2) fetch the title of example.org.

        

      


      
04 Skills system

      

        
⚡

        

          
Save a workflow with create_skill. Later, just say the example prompt and Warden activates it. Manage with list_skills / activate_skill / deactivate_skill.

        

      


      
05 What works best

      

        
→
Be specific. Include the file path, URL, or input data.

        
⇄
Parallelize. "X AND Y in parallel" — independent tasks run concurrently.

        
!
Read BLOCKED messages. If a tool fails, Warden says BLOCKED — don't retry blindly.

        
●
Watch the verbose bar. Below the composer — shows what Warden is doing right now.

      


      
06 Settings worth knowing

      

        
Orchestrator
The model that replies to you. Pick a strong one.

        
Atlas / Artemis
Research and review sub-agent model.

        
Tools
Iris / Byte / Dexter execution. Use a small local model.

        
Council Seats
Skeptic / Pragmatist / Synthesist. Different models = different perspectives. Blank inherits Atlas.

        
num_ctx overrides
Orchestrator / Atlas / Toolcall. Blank = model default.

        
Thinking
On / Max / Off. Global default, no per-chat override.

        
Ollama URL
Local daemon endpoint. Models populate all dropdowns.

      


      
07 Audit

      

        
✓

        

          
The Run audit button (left rail, bottom) kicks off 100 scenarios through the chat API. 15–45 min, Warden is busy the whole time. Log at /tmp/audit-100-run.log — paste it into Warden to fix, or use Claude Code if Warden is broken.

⚠

        

          
    
            
  • Don't run Warden as admin.
    • 
            
  • Don't give it write access to important things — email, prod repos, keys.
    • 
            
  • Make backups before letting it touch anything you care about.
    • 
            
  • Don't run this on your only or work computer.

      Run Audit
      
    

    

      
Runs 100+ scenarios against Warden through the chat API — conversation, file ops, bash, skills, The Council, parallel Atlas, sub-agents, monitor, blocked paths, voice output, injection guards.

      
This takes 15–45 minutes. Warden will be busy the whole time; you won't be able to chat normally until the audit finishes.

      
When done, the log is at /tmp/audit-100-run.log. To fix failures:

      
    
        
  1. In Warden chat paste: cat /tmp/audit-100-run.log — then ask Warden to go through it, fix issues, and report back.
    2. 
        
  2. If Warden itself is broken, run a console agent runner (Claude Code) in the dockbox directory and paste the audit log there.
